## What This Assessment Covers

Combined static and dynamic examination of a widely-used Android health-tracking application. Static analysis characterizes the application's structure, declared permissions, third-party SDK architecture, and embedded resource strings. Dynamic analysis captures live network behavior on a real device — producing an actual traffic profile rather than an emulated approximation — and validates consistency with the static findings.

---

## Methodology

**Static Analysis:**
APK transferred into a controlled Windows 11 VM and opened in JADX-GUI for source code and manifest review. Permissions, declared components, intent filters, and third-party SDK references documented from `AndroidManifest.xml`. Targeted searches in JADX for URLs, network endpoints, and external service references. Apktool used for raw resource and file structure inspection.

**Dynamic Analysis:**
Physical Android device used for execution — providing accurate native-library behavior and avoiding ABI translation artifacts introduced by ARM emulators on Apple Silicon. PCAPdroid configured for per-application traffic isolation, capturing all outbound DNS, HTTPS, and QUIC connections generated by PeriodTracker during realistic initial-load interaction (startup screens, content navigation). All captured traffic inspected through per-connection summaries.

**Correlation:** Static declarations validated against observed runtime behavior across both phases.

## Static Analysis Findings

### Permissions (AndroidManifest.xml)
| Category | Permissions |
|---|---|
| **Network** | `INTERNET`, `ACCESS_NETWORK_STATE`, `ACCESS_WIFI_STATE` |
| **System** | `WAKE_LOCK`, `RECEIVE_BOOT_COMPLETED`, `VIBRATE` |
| **Storage** | `READ_EXTERNAL_STORAGE`, `WRITE_EXTERNAL_STORAGE` |
| **Location** | `ACCESS_FINE_LOCATION`, `ACCESS_COARSE_LOCATION` |
| **Accounts** | `GET_ACCOUNTS` |
| **Messaging/Push** | GCM / C2D_MESSAGE, install referrer |

### Third-Party SDK Architecture
- **Analytics & Attribution:** Firebase, Flurry, Adjust, AppsFlyer, Segment
- **Crash Reporting:** Crashlytics / Fabric
- **Consent Management:** Usercentrics (GDPR/CMP)
- **Social:** Facebook SDK
- **Fitness Integrations:** Fitbit API, Google Fit (optional, user-activated)
- **Backend:** Firebase Realtime Database (`https://flo-health.firebaseio.com`)

### Key Static Observations
- No hardcoded custom REST endpoints beyond the Firebase backend
- All identified endpoints use HTTPS — no weakened network security configuration
- Intent filters define custom deep links for pregnancy mode and content navigation
- Multiple activities for user registration, onboarding, health tracking, surveys, and debug/testing screens

## Dynamic Analysis Findings

### Traffic Profile (Live Capture)
| Domain | Service | Protocol |
|---|---|---|
| `flo.health`, `api.owhealth.com`, `content.owhealth.com` | Flo Health backend + content | HTTPS |
| `crashes.owhealth.com` | Crash reporting | HTTPS |
| `web.cmp.usercentrics.eu`, `v1.api.service.cmp.usercentrics.eu`, `app.usercentrics.eu` | Consent management (Usercentrics) | HTTPS |
| `firebaselogging.googleapis.com` | Firebase analytics logging | HTTPS |
| `www.googletagmanager.com` | Google Tag Manager | HTTPS |
| `bat.bing.com` | Microsoft/Bing tracking | HTTPS |
| `customer-[id].cloudflarestream.com` | CDN / media delivery | HTTPS |
| `fonts.gstatic.com` | Google font assets | HTTPS |

### Notable Observations
- **All traffic encrypted** — HTTPS (TLS 443) and QUIC (UDP). No unencrypted HTTP observed.
- **Pre-launch consent requests:** Usercentrics CMP initiated requests before any user interaction. This pre-consent initialization pattern warrants attention in privacy and regulatory contexts — GDPR and similar frameworks may require affirmative consent prior to any third-party data transmission.
- **Static/dynamic consistency:** Every domain observed at runtime corresponds directly to an SDK, endpoint, or service identified during static analysis. No unexpected or undeclared third-party servers detected.
- **Runtime permissions:** Notification, activity/motion, and background sync prompts matched manifest declarations exactly — no unauthorized runtime permission requests.
